# Workflow: Repo Scan — V9.3
# Trigger: applying workflow to an existing project

## Phase 0 — Pre-flight conflict check

Before touching anything, check if `.ai/` already exists:

```bash
ls .ai/ 2>/dev/null && echo "EXISTS" || echo "FRESH"
```

**If FRESH:** skip to Phase 1.

**If EXISTS — output conflict report:**

```
## Conflict report

Will REPLACE (standard files — safe):
  .ai/workflows/*.md
  .ai/routing.md

Will MERGE (project content preserved):
  .ai/AGENTS.md  ← keep project constraints, update standard sections

Will GENERATE FRESH:
  .ai/SKILLS-TODO.md

Will NOT touch (never modified by workflow):
  .ai/tasks/**      ← <N> task files found
  .ai/memory/**     ← <N> memory files found
  .ai/skills/**     ← <N> skill files found (append only if new)
  docs/**           ← source of truth, never touched
  .ai/module-map.md ← project-specific

Proceed? (human confirms before any file is written)
```

Stop and wait for human confirmation before Phase 1.

## Phase 1 — Lightweight discovery (token-efficient)

```bash
# Project type detection
find . -maxdepth 2 \
  -name "package.json" -o -name "*.csproj" -o \
  -name "pyproject.toml" -o -name "go.mod" -o \
  -name "Gemfile" -o -name "pubspec.yaml" \
  | grep -v node_modules | grep -v .git | sort

# App boundaries
ls apps/ packages/ services/ src/ themes/ 2>/dev/null

# Entry points
find . -maxdepth 4 \
  -name "main.ts" -o -name "index.ts" -o -name "Program.cs" -o \
  -name "main.py" -o -name "app.py" -o -name "server.ts" \
  | grep -v node_modules | grep -v dist | grep -v .git | head -20

# Schema / DB
find . -maxdepth 5 \
  -name "*.prisma" -o -name "schema.sql" -o \
  -name "*.edmx" -o -name "DbContext*.cs" \
  | grep -v node_modules | grep -v .git | head -10
```

Hard limit: read at most 5 files per detected app, max 80 lines each.
Do NOT read: test files, dist/, node_modules/, .git/, bin/, obj/.

| `next`, `react` in deps | Frontend=Next.js/React |
| `jest`, `vitest`, `xunit`, `pytest` | Test framework |
| `jsonwebtoken`, `passport`, `JwtBearer` | Auth=JWT |
| `prisma`, `typeorm`, `EntityFramework` | ORM |
| `tailwind` | CSS=Tailwind |

Mark ✅ for detected, ❓ for unknown.

## Phase 5 — AGENTS.md merge (existing project only)

If AGENTS.md exists:
1. Read existing file
2. Identify project-specific sections (constraints, auth, build commands)
3. Keep those sections verbatim
4. Replace standard sections (golden rules, output format, pre-coding read order)
5. Write merged result

If AGENTS.md does not exist: generate from template.
